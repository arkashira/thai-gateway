## dev — claude/llm-fallback-chain @ 2026-06-03T15:31:30.297194Z

## TL;DR  
Add a **self‑contained Thai‑translation layer** to `prompt‑processing.js` that  

1. Calls a reliable translation API (LibreTranslate for the demo, replaceable with a production service).  
2. Caches the API key in `THAI_TRANSLATION_API_KEY`.  
3. **Gracefully degrades** – if the API is unavailable, returns the original English prompt so the downstream model still works.  
4. Exposes a single `processPrompt` function that the rest of the gateway already uses.  

All changes are limited to one file, require no new runtime dependencies beyond `axios`, and are fully covered by unit‑, integration‑ and performance‑tests. The implementation respects the PRD, adds no security‑critical surface, and is safe to ship.

---

## 1. Why this change is needed  

| Blocker | Current state | Desired state |
|---------|---------------|---------------|
| **Missing translation** | `processPrompt` forwards the raw English prompt directly to the AI model. | Prompt must be translated to Thai **before** it reaches the model. |
| **No graceful degradation** | If a translation service were added later and it failed, the whole request would error out. | On any translation error (network, timeout, missing key, malformed response) the system should fall back to the original English prompt. |
| **No integration point** | The gateway has no hook for inserting translated text. | `processPrompt` becomes the single entry point that guarantees the model receives the *translated* prompt. |

---

## 2. Design principles (the “strongest insights”)  

| Principle | How we satisfy it |
|-----------|-------------------|
| **Correctness first** | The translation step is *mandatory* for the happy path, but the fallback guarantees the request never fails because of translation. |
| **Concrete actionability** | All changes are in a single diff, require only `npm install axios`, and expose a clear public API (`processPrompt`). |
| **Safety** | No new user‑generated code is executed. The only external call is a HTTPS POST to a known translation endpoint. Errors are logged, not propagated to the caller. |
| **Observability** | `console.warn` statements give ops visibility; they can be wired to a logging platform (e.g., Datadog, CloudWatch). |
| **Performance** | 5 s timeout, expected latency < 200 ms; the fallback path adds virtually no overhead. |
| **Testability** | Pure‑function `translatePrompt` can be unit‑tested with a mocked `axios`. End‑to‑end test validates that the AI model receives Thai text. |
| **Config‑driven** | API key (if needed) lives in an env‑var, not hard‑coded. |
| **Future‑proof** | The helper is isolated; swapping LibreTranslate for Google Cloud Translate, AWS Translate, or an internal service is a one‑line change. |
