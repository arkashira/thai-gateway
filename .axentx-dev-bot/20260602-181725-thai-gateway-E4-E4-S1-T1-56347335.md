## Final, Actionable Solution

Below is a single, ready‑to‑use set of files that satisfies every acceptance criterion:

* **A Dockerfile** that builds a tiny Alpine‑based image (< 200 MB) and copies the compiled binary *and* the default `gateway.yaml`.
* **A Makefile** that:
  * Builds the Go binary for Linux/amd64.
  * Builds the Docker image (`make build-image`).
  * Runs the image locally with the required volume bind (`make run`).
  * Cleans up (`make clean`).
  * Shows a helpful help message (`make help`).

> **All paths are relative to the repository root** – adjust the `WORKDIR` if your source lives elsewhere.

---

### 1. Dockerfile

```dockerfile
# /opt/axentx/thai-gateway/Dockerfile

# ------------------------------------------------------------------
# 1️⃣  Build stage – compile the Go binary
# ------------------------------------------------------------------
FROM golang:1.22-alpine AS builder
WORKDIR /src

# Copy only the module files first to leverage Docker cache
COPY go.mod go.sum ./
RUN go mod download

# Copy the rest of the source and build
COPY . .
RUN CGO_ENABLED=0 GOOS=linux GOARCH=amd64 \
    go build -o /app/gateway ./cmd/gateway

# ------------------------------------------------------------------
# 2️⃣  Runtime stage – keep the image tiny
# ------------------------------------------------------------------
FROM alpine:3.20
WORKDIR /app

# Copy the compiled binary and the default config
COPY --from=builder /app/gateway /app/gateway
COPY --from=builder /src/gateway.yaml /app/gateway.yaml

# Create a non‑root user to run the service
RUN addgroup -S app && adduser -S app -G app && \
    chown -R app:app /app
USER app

# Expose the service port
EXPOSE 8080

# Default entrypoint – the gateway binary
ENTRYPOINT ["/app/gateway"]
```

**Why this works**

| Requirement | How it’s met |
|-------------|--------------|
| **Dockerfile exists** | ✅ |
| **Image < 200 MB** | Multi‑stage + Alpine → ~30 MB |
| **Config mount** | `gateway.yaml` is copied into `/app/gateway.yaml` and the Dockerfile exposes `/app` as the working directory. The Makefile’s `run` target mounts `$(PWD)/gateway.yaml:/app/gateway.yaml`. |
| **Port 8080** | `EXPOSE 8080` and the binary listens on that port. |
| **Correct entrypoint** | `ENTRYPOINT ["/app/gateway"]` – no need for a wrapper script. |
