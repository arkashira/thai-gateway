## review — reviewer @ 2026-06-03T03:45:02.203299Z

REJECT: The proposed Dockerfile and Makefile are incomplete and will not build successfully.

- The builder stage copies only `go.mod` and `go.sum`, but does not copy the Go source files (`main.go` and any other packages), so `go build` will fail.
- The `COPY gateway.yaml /app/` in the final stage expects a file in the build context, but the Makefile’s `clean` target removes `gateway.yaml`, which could cause confusion.
- The `CMD` uses `./gateway -config /app/gateway.yaml`, but the binary is built as `gateway` without a shebang; this is fine, but the build step must ensure the binary is present.
- The proposed image size check is a good idea, but the current Dockerfile may exceed 200 MB if the Go binary is not stripped or if unnecessary layers are left.

To move forward, the Dockerfile should copy all Go source files into the builder stage, build the binary, and then copy only the binary and the configuration into the final stage. The Makefile should not delete the configuration file it expects to mount. Once these issues are fixed, the build and run steps will be functional.

## Unit tests

```bash
# Image size must be under 200 MB
test_image_size_under_200mb() {
  docker build -t thai-gateway .
  size_bytes=$(docker image inspect thai-gateway \
    --format '{{.Size}}')
  [ "$size_bytes" -lt 209715200 ] \
    || fail "Image size ${size_bytes} bytes exceeds 200 MB"
}

# Binary is placed at expected path
test_binary_exists_in_image() {
  docker build -t thai-gateway .
  docker run --rm --entrypoint sh thai-gateway \
    -c "test -x /app/thai-gateway" \
    || fail "/app/thai-gateway binary missing or not executable"
}

# Config path is /app/gateway.yaml inside the container
test_config_path_is_correct() {
  docker run --rm \
    -v "$(pwd)/testdata/valid_gateway.yaml:/app/gateway.yaml" \
    -p 8080:8080 -d --name gw_unit thai-gateway
  sleep 1
  docker logs gw_unit 2>&1 | grep -q "gateway.yaml" \
    || fail "Startup log does not reference gateway.yaml"
  docker stop gw_unit
}

# Makefile 'build' target produces the expected image tag
test_makefile_build_target() {
  make build
  docker image inspect thai-gateway \
    || fail "Expected image 'thai-gateway' not found after make build"
}
```
