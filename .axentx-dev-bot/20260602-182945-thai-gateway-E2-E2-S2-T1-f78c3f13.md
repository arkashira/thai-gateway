## 1. Acceptance Criteria (measurable)

| # | Criterion | Measurement |
|---|-----------|-------------|
| 1 | **Provider Selection** | The router returns the provider ID that has the highest remaining quota among all healthy providers. |
| 2 | **Quota Exhaustion** | When every provider’s remaining quota is `0`, the router responds with HTTP status **429 Too Many Requests** and includes a non‑empty `Retry-After` header. |
| 3 | **Health Check** | Only providers marked as *healthy* (e.g., `HealthStatus == "UP"`) are considered for selection. |
| 4 | **Audit Logging** | Each routing decision (provider chosen or 429 emitted) is logged with timestamp, request ID, selected provider ID (or `NONE`), and remaining quotas of all providers. |
| 5 | **Determinism** | Given the same provider state, the router always selects the same provider (no randomization). |
| 6 | **Thread Safety** | Concurrent routing requests do not cause race conditions or inconsistent quota reads. |
| 7 | **Performance** | Routing decision latency is < 5 ms under normal load (verified in integration test). |
