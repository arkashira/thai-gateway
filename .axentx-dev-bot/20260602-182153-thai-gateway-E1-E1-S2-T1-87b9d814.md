## 1. Acceptance Criteria  

1. **Quota Persistence** – For each provider the gateway stores a numeric `remaining_quota` in Redis under the key `quota:{providerId}` with a TTL equal to the provider’s quota window (seconds).  
2. **Quota Decrement** – Each successful request atomically decrements `remaining_quota` (Redis `DECRBY`) and never drops below 0.  
3. **Quota Check Before Routing** – The gateway reads `remaining_quota`; if the value is > 0 the request is forwarded to that provider.  
4. **Provider Rotation** – If the current provider’s quota is 0, the gateway iterates the configured provider list (round‑robin) and selects the first provider with a positive quota.  
5. **All Providers Exhausted** – When every provider’s quota is 0, the gateway responds with **HTTP 429 Too Many Requests** and a `Retry-After` header set to the smallest remaining TTL among all provider quota keys.  
6. **Atomicity & Race‑Safety** – All read‑modify‑write operations on Redis are performed atomically (Lua script or `MULTI/EXEC`).  
7. **Redis Failure Fallback** – If Redis is unreachable, the request proceeds using an in‑memory fallback store for the lifetime of that request and logs a warning; the fallback does **not** persist across requests.

## 2. Unit Tests (Jest – Node.js)

```js
// src/services/quota.test.js
const Redis = require('ioredis-mock');          // in‑memory mock that supports Lua & TTL
const { QuotaStore } = require('./quota');      // implementation under test

describe('QuotaStore (unit)', () => {
  let redisClient;
  let quotaStore;

  beforeEach(() => {
    redisClient = new Redis();
    quotaStore = new QuotaStore(redisClient);
  });

  afterEach(async () => {
    await redisClient.flushall();
    await redisClient.disconnect();
  });

  test('setQuota stores value with correct TTL', async () => {
    await quotaStore.setQuota('provA', 100, 60); // 60 s window
    const ttl = await redisClient.ttl('quota:provA');
    const stored = await redisClient.get('quota:provA');

    expect(ttl).toBeGreaterThanOrEqual(55); // allow small drift
    expect(ttl).toBeLessThanOrEqual(60);
    expect(stored).toBe('100');
  });

  test('decrement reduces quota atomically and never goes negative', async () => {
    await quotaStore.setQuota('provB', 2, 30);
    await quotaStore.decrement('provB'); // → 1
    await quotaStore.decrement('provB'); // → 0
    await quotaStore.decrement('provB'); // should stay 0

    const final = await redisClient.get('quota:provB');
    expect(final).toBe('0');
  });

  test('getQuota returns null for unknown provider', async () => {
    const val = await quotaStore.getQuota('unknown');
    expect(val).toBeNull();
  });

  test('concurrent decrements are safe (simulated with Promise.all)', async () => {
    await quotaStore.setQuota('provC', 5, 30);
    const promises = Array.from({ length: 5 }, () => quotaStore.decrement('provC'));
    await Promise.all(promises);
    const remaining = await redisClient.get('quota:provC');
    expect(remaining).toBe('0');
  });

  test('fallback to in‑memory store when Redis throws', async () => {
    // Force Redis client to throw on any command
    redisClient.get = jest.fn(() => { throw new Error('Redis down'); });
    redisClient.set = jest.fn(() => { throw new Error('Redis down'); });

    const fallback = await quotaStore.getQuota('provD');
    expect(fallback).toBeNull(); // in‑memory store empty for this request

    // setQuota should store in the in‑memory map
    await quotaStore.setQuota('provD', 10, 60);
    const memVal = quotaStore._memoryStore.get('provD'); // internal for test only
    expect(memVal.remaining).toBe(10);
  });
});
```
