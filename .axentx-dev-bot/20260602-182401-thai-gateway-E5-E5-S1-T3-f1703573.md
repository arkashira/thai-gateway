## 1. Acceptance Criteria  
- **Prometheus endpoint**: `GET /metrics` returns a `text/plain` body containing at least the metrics `gateway_requests_total`, `gateway_errors_total`, `gateway_response_time_seconds`, and `gateway_provider_usage{provider="…"}`
- **Metric values**:  
  - `gateway_requests_total` increments by 1 for each incoming request.  
  - `gateway_errors_total` increments by 1 for each response with status >= 400.  
  - `gateway_response_time_seconds` records the latency of each request (histogram or summary).  
  - `gateway_provider_usage{provider="…"}` increments per provider used.  
- **Logging**:  
  - Every request is logged with a unique `requestId` (UUID v4).  
  - Log format: `"[<timestamp>] <level> requestId=<id> method=<method> path=<path> status=<status> duration=<ms>ms provider=<provider>"`  
  - Logs are emitted to console (stdout) and to a file `gateway.log`.  
- **Traceability**: The `requestId` is propagated through the request lifecycle and appears in both the log entry and the Prometheus metrics (e.g., as a label if desired).
