## 1. Acceptance Criteria
- **Retry Count**: The gateway must retry a failed request to the same provider up to **3 times** before considering it a permanent failure.
- **Fail‑over Routing**: If all retries to the current provider fail, the request must be routed to the **next available provider** in the configured list.
- **Error Logging**: Every failed request (after retries) must be logged with:
  - Timestamp
  - Provider ID
  - Request details (method, URL, headers, body)
  - Error code/message
  - Retry count
- **Log Persistence**: Logs must be written to the configured log file (`/opt/axentx/thai-gateway/logs/error.log`) and be retrievable via the logger API.
- **No Duplicate Logs**: A single failed request should generate **exactly one** log entry, regardless of retry attempts.
