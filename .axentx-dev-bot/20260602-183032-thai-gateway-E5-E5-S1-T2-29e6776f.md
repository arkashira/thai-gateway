**Test Plan – Thai Gateway Platform‑Specific Rate Limiting**

---

### 1. Acceptance Criteria
1. **API‑Key Generation** – On the first registration request, the system returns a newly generated API key that is stored and associated uniquely with the requesting platform.
2. **Key Scope Isolation** – An API key can only access resources belonging to its own platform; any request using a key for another platform must be rejected with HTTP 401.
3. **Per‑Platform Rate Limit** – Each platform is limited to **X requests per minute** (configurable, default 120 rpm). Exceeding the limit returns HTTP 429 with a `Retry-After` header.
4. **Independent Quotas** – Rate limits are enforced per platform; heavy traffic from Platform A must not affect the quota of Platform B.
5. **Rate‑Limit Reset** – After the time window expires, the quota resets and the platform can make requests again.
6. **Documentation** – The README contains a section “Onboarding & Rate Limiting” that describes the registration endpoint, how to retrieve the API key, and the default rate‑limit values.
