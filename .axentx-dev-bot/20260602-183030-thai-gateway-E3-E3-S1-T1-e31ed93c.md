## 1. Acceptance Criteria

1. **AC1 — Schema conformance:** `formatOpenAI(payload)` returns an object containing `model`, `messages[]`, and optional `temperature`/`max_tokens`; `formatAnthropic(payload)` returns `model`, `messages[]` (or `prompt`), `max_tokens`, and `anthropic_version`.
2. **AC2 — Required-field validation:** Missing `model` or `messages` for OpenAI throws `ValidationError`; missing `model` or (`messages`/`prompt`) for Anthropic throws `ValidationError`.
3. **AC3 — Malformed-request logging:** Every validation failure emits a structured log entry with `timestamp` (ISO-8601), `provider`, `error`, and `payload` (sanitized).
4. **AC4 — Multi-provider support:** At least 2 distinct provider formats (OpenAI, Anthropic) are exported and independently callable.
4. **AC5 — Idempotency:** Calling a formatter twice on the same input yields structurally equal outputs (no mutation of input).
5. **AC6 — Latency budget:** Formatter + validation completes in < 5 ms p95 for payloads ≤ 100 messages.
6. **AC7 — No PII leakage:** Sanitized payload in logs must redact fields matching `/(api[_-]?key|token|password|email)/i`.
