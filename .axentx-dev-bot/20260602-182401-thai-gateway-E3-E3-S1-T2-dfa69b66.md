# Diagnosis

The current implementation in `/opt/axentx/thai-gateway/src/middleware/headers.js` lacks the functionality to add rate limit headers to responses. The middleware responsible for handling responses does not check or forward rate limit headers from the providers.

# Proposed Change

We will modify the existing response middleware to check for rate limit headers in the provider's response and forward them to the client if present. The rate limit headers to be forwarded are `X-RateLimit-Limit`, `X-RateLimit-Remaining`, and `X-RateLimit-Reset`.

# Implementation

```javascript
// Before (in /opt/axentx/thai-gateway/src/middleware/headers.js)
const responseHeadersMiddleware = (req, res, next) => {
  // ... other headers
  next();
};

// After
const responseHeadersMiddleware = (req, res, next) => {
  // ... other headers

  // Check if rate limit headers are present in the provider's response
  if (req.providerResponse && req.providerResponse.headers) {
    const rateLimitHeaders = ['x-ratelimit-limit', 'x-ratelimit-remaining', 'x-ratelimit-reset'];

    rateLimitHeaders.forEach((header) => {
      if (req.providerResponse.headers[header]) {
        res.setHeader(header.replace(/^x-/, '').replace(/-/g, ' '), req.providerResponse.headers[header]);
      }
    });
  }

  next();
};
```
