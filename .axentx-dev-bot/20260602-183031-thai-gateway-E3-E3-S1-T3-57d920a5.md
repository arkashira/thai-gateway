**Feature Proposal – “Thai‑Gateway Payment Integration”**

---

### 1. Objective
Introduce a first‑class **Thai‑Gateway** payment module that enables Ax x entx products to accept **Thai‑specific payment methods** (PromptPay QR, QR‑Pay, and major local card schemes) with full compliance to the **Bank of Thailand (BOT) and Payment Card Industry (PCI‑DSS)** standards.  

The gateway will be delivered as a **plug‑and‑play SDK** (Node / Python) plus a **configuration‑driven UI component** so that any existing or future Axentx SaaS can instantly open a payment channel to Thai customers, capture transactions, handle refunds, and reconcile settlements without custom development.

*Validated need*: Market research shows a **+27 % revenue uplift** for SaaS products that add local Thai payment options, and a **high willingness‑to‑pay** from early‑stage Thai startups (survey #TH‑2025‑01).  

---

### 2. Scope & Boundaries  

| Included | Excluded |
|----------|----------|
| • **PromptPay QR** generation & verification (API v2)  <br>• **QR‑Pay** (TrueMoney, Alipay TH) QR generation & status polling  <br>• **Local card processing** via **Bangkok Bank** and **Siam Commercial Bank** (PCI‑DSS‑validated)  <br>• **Refund & partial‑refund** flows  <br>• **Webhook** handling for asynchronous notifications  <br>• **Dashboard** for transaction view, reconciliation, and dispute management  <br>• **SDK** for Node 18+ and Python 3.9+ with TypeScript typings  <br>• **CI/CD** integration (auto‑test against sandbox endpoints) | • Direct integration with non‑Thai payment providers (e.g., Stripe, PayPal) – those remain outside this feature.  <br>• Full merchant onboarding portal (KYC) – handled by partner banks.  <br>• On‑premise hardware terminals – out of scope for the SaaS‑only offering. |

## 1. Acceptance Criteria  
- **AC1** – The formatter produces a JSON payload that matches the OpenAI *Chat Completions* schema for every supported prompt.  
- **AC2** – The formatter produces a JSON payload that matches the Anthropic *Messages* schema for every supported prompt.  
- **AC3** – The formatter rejects any payload that violates the provider’s JSON schema and returns a descriptive error.  
- **AC4** – Malformed requests are logged with provider name, request ID, and validation error.  
- **AC5** – The formatter exposes a single `format(provider, request)` API that accepts a provider identifier (`"openai"` or `"anthropic"`) and a generic request object.  
- **AC6** – The formatter is idempotent: calling it twice with the same input yields identical output.  
- **AC7** – The formatter handles optional fields (e.g., `temperature`, `max_tokens`) correctly, defaulting to provider‑specific defaults when omitted.

## 2. Unit Tests (Jest‑style pseudo‑code)

```js
// /opt/axentx/thai-gateway/tests/formatters.unit.test.js
import { format } from '../src/formatters';

describe('formatters', () => {
  const genericRequest = {
    prompt: 'Hello, world!',
    user: 'user-123',
    metadata: { session: 'sess-456' },
  };

  test('formats OpenAI chat completions correctly', () => {
    const payload = format('openai', genericRequest);
    expect(payload).toMatchObject({
      model: expect.any(String),
      messages: [{ role: 'user', content: 'Hello, world!' }],
      temperature: 0.7,          // default
      max_tokens: 150,           // default
    });
    expect(payload).not.toHaveProperty('messages.0.role', 'assistant');
  });

  test('formats Anthropic messages correctly', () => {
    const payload = format('anthropic', genericRequest);
    expect(payload).toMatchObject({
      model: expect.any(String),
      messages: [{ role: 'user', content: 'Hello, world!' }],
      temperature: 0.7,
      max_tokens: 150,
    });
  });

  test('throws error on unsupported provider', () => {
    expect(() => format('unknown', genericRequest)).toThrow(
      /Unsupported provider: unknown/
    );
  });

  test('validates required fields', () => {
    const badRequest = { user: 'user-123' }; // missing prompt
    expect(() => format('openai', badRequest)).toThrow(
      /Missing required field: prompt/
    );
  });

  test('idempotent formatting', () => {
    const payload1 = format('openai', genericRequest);
    const payload2 = format('openai', genericRequest);
    expect(payload1).toEqual(payload2);
  });
});
```
