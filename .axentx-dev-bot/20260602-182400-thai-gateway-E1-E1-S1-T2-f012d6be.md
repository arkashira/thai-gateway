## 1. Acceptance Criteria
- **AC1** – The `/health` HTTP GET endpoint returns a JSON payload with a `status` field (`"healthy"` or `"unhealthy"`).  
- **AC2** – The payload includes a `providers` array where each element contains `name`, `responseTimeMs`, `errorRatePct`, and `congestionScore` (computed as `responseTimeMs * errorRatePct`).  
- **AC3** – The provider with the lowest `congestionScore` is flagged as `primary: true`; all others have `primary: false`.  
- **AC4** – If any provider’s `errorRatePct` ≥ `ERROR_RATE_THRESHOLD` (env var, default = 20) the endpoint returns HTTP 503 and `status: "unhealthy"`.  
- **AC5** – The health‑check logic reads configuration (`PROVIDER_LIST`, `ERROR_RATE_THRESHOLD`, `RETRY_COUNT`) from environment variables at runtime.  
- **AC6** – A failed request to the primary provider is automatically retried up to `RETRY_COUNT` times on the next‑best provider before returning an error to the caller.  
- **AC7** – All response times and error rates are sourced from the in‑memory `metricsStore` which is updated by the existing request‑proxy layer; the health check must not mutate this store.
