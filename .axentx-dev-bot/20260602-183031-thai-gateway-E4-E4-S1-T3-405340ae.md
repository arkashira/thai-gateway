## 1. Acceptance Criteria
1. **HTTP 200 response** – `GET /health` returns status 200 within **50 ms** for **99 %** of requests measured over a 1‑minute window.  
2. **JSON payload** – Body is valid JSON containing:
   - `status: "healthy"`  
   - `timestamp` in **ISO‑8601 UTC** format (`YYYY‑MM‑DDTHH:mm:ss.sssZ`)  
   - `gatewayVersion` that matches the version field in `package.json`.  
3. **Unauthenticated access** – The endpoint can be called without any authentication headers, cookies, or tokens.  
4. **Prometheus metric** – Each successful health‑check increments the gauge `thai_gateway_health_check_success_total` exposed on the existing `/metrics` endpoint.  
5. **Request logging** – Every health‑check request produces a log entry via the gateway logger with fields: `method`, `path`, `timestamp`, optional `requestId` (if present in headers), and `responseTimeMs`.  
6. **Rate‑limit exhaustion alert** – If any provider’s rate‑limit counter is **0**, the health response includes an additional field `alerts` that is an array containing strings of the form `rate_limit_exhausted:<providerId>`.  
7. **Graceful shutdown** – After the process receives **SIGTERM**, the `/health` endpoint continues to return HTTP 200 (with the same payload) until the server fully exits, allowing orchestrators to confirm shutdown.
