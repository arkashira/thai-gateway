## Description
This feature introduces a dynamic system to adjust the crawl budget based on the performance of each data source across the three weakest quality dimensions identified: `severity_accuracy`, `audience_specificity`, and `reproducibility`. By prioritizing high-signal sources, we enhance the accuracy and relevance of the data used for product validation, ensuring that the pipeline synthesizes products that solve validated paying needs more effectively.

## Implementation
1. **Scoring Model**: Develop a machine learning model (using pgvector) that evaluates each data source's contribution to the three weak dimensions. The model will calculate a "signal strength" score for each source per dimension.
2. **Real-time Crawl Budget Adjustment**: Integrate the scoring model with the existing crawl budget management system. The system will dynamically increase the crawl budget for sources with high signal strength and decrease it for low-performing sources.
3. **Dashboard**: Create a monitoring dashboard to visualize the impact of source prioritization on quality metrics over time.

- Enhances the accuracy of product specs by leveraging high-signal sources.
- Extends the portfolio by enabling the synthesis of products that address validated needs more effectively.

## Dependencies

- Existing crawl budget management infrastructure.

## Testing

- Integration tests for crawl budget adjustments.
- Validation against historical data to confirm improved performance metrics.

### 1. Acceptance Criteria
1. **Correct Translation** – For a given English prompt, the middleware returns the accurate Thai translation (≥95% BLEU score against a reference translation) before forwarding the request.
2. **Intent Preservation** – Semantic similarity between original English prompt and Thai translation (measured by cosine similarity ≥0.90 using a multilingual sentence encoder) must be maintained.
3. **Latency SLA** – 90 % of translated requests add ≤ 100 ms overhead compared to the baseline (no‑translation) path.
4. **Graceful Fallback** – If the translation service returns an error or times‑out, the original English prompt is forwarded unchanged.
5. **Error Logging** – Translation failures are logged with request ID, error type, and timestamp.
6. **Observability** – Metrics `translation_success_total`, `translation_failure_total`, and `translation_latency_ms` are exposed via Prometheus.
7. **Idempotence** – Re‑processing a request that already contains a `X-Translated: true` header does not trigger a second translation.

package middleware_test

import (
    "bytes"
    "context"
    "net/http"
    "net/http/httptest"
    "testing"
    "time"

    "github.com/stretchr/testify/assert"
    "github.com/stretchr/testify/require"
    "github.com/axentx/thai-gateway/src/proxy/middleware"
)

// mock translation service
type fakeTranslator struct {
    resp  string
    err   error
    delay time.Duration
}

func (f *fakeTranslator) Translate(ctx context.Context, eng string) (string, error) {
    if f.delay > 0 {
        select {
        case <-time.After(f.delay):
        case <-ctx.Done():
            return "", ctx.Err()
        }
    }
    return f.resp, f.err
}

func TestTranslateMiddleware_Success(t *testing.T) {
    tr := &fakeTranslator{resp: "สวัสดี", err: nil}
    mw := middleware.NewTranslate(tr)

    req := httptest.NewRequest(http.MethodPost, "/ai", bytes.NewBufferString(`{"prompt":"Hello"}`))
    rec := httptest.NewRecorder()

    // downstream handler just echoes body
    downstream := http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        body, _ := io.ReadAll(r.Body)
        w.Write(body)
    })

    mw.Handler(downstream).ServeHTTP(rec, req)

    // assert body was translated
    assert.Equal(t, `{"prompt":"สวัสดี"}`, rec.Body.String())
    // header indicating translation
    assert.Equal(t, "true", rec.Result().Header.Get("X-Translated"))
}

func TestTranslateMiddleware_FallbackOnError(t *testing.T) {
    tr := &fakeTranslator{resp: "", err: errors.New("service down")}
    mw := middleware.NewTranslate(tr)

    req := httptest.NewRequest(http.MethodPost, "/ai", bytes.NewBufferString(`{"prompt":"Hello"}`))
    rec := httptest.NewRecorder()

    downstream := http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        body, _ := io.ReadAll(r.Body)
        w.Write(body)
    })

    mw.Handler(downstream).ServeHTTP(rec, req)

    // original prompt forwarded unchanged
    assert.Equal(t, `{"prompt":"Hello"}`, rec.Body.String())
    // no translation header
    assert.Empty(t, rec.Result().Header.Get("X-Translated"))
}

func TestTranslateMiddleware_LatencyWithinSLA(t *testing.T) {
    tr := &fakeTranslator{resp: "สวัสดี", err: nil, delay: 50 * time.Millisecond}
    mw := middleware.NewTranslate(tr)

    start := time.Now()
    req := httptest.NewRequest(http.MethodPost, "/ai", bytes.NewBufferString(`{"prompt":"Hello"}`))
    rec := httptest.NewRecorder()
    downstream := http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) { w.WriteHeader(http.StatusOK) })

    mw.Handler(downstream).ServeHTTP(rec, req)
    elapsed := time.Since(start)

    assert.LessOrEqual(t, elapsed.Milliseconds(), int64(100), "translation overhead >100ms")
}
