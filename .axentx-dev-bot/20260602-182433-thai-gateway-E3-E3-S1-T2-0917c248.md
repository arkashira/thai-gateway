### Diagnosis & Requirements
The system requires a **translation middleware** to be placed in `src/proxy/middleware.go`. This middleware must:
1.  **Intercept** all incoming requests at the gateway level.
2.  **Translate** the `prompt` field in the JSON body from English to Thai **before** forwarding the request.
3.  **Preserve** the original request structure, replacing only the `prompt` string.
4.  **Meet Latency SLA:** Add < 100ms latency to 90% of requests. This is achieved by using a fast in-process stub (simulating ~20ms) and setting a tight context deadline (80ms).
5.  **Fallback:** If translation fails (error or timeout), the original request body must be used without modification.

### Proposed Change

1.  **`TranslationService` Interface:** An abstraction layer to allow swapping the stub for a real API client later.
2.  **`googleTranslateStub`:** A lightweight, deterministic implementation that adds a 20ms delay and prefixes the text with `[TH]`.
3.  **`translationMiddleware`:** The core logic that reads the body, translates the prompt, handles errors, and forwards the request.
4.  **`NewTranslationMiddleware`:** A factory function to wire the service into the HTTP router.

	assert.NotNil(t, res)
	assert.Equal(t, req.Prompt, res.Response)
	assert.Less(t, time.Since(start), 100*time.Millisecond)

	// Test cancellation
