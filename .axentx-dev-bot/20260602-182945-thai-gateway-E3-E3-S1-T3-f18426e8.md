**3. Integration Tests**

- **Happy Paths:**
  - A Thai prompt is sent to the gateway, and the middleware translates it to English before passing it to the AI provider.
  - An English prompt is sent to the gateway, and the middleware passes it as is to the AI provider.
- **Edge Cases:**
  - An empty prompt is sent to the gateway, and the middleware handles it gracefully without crashing or causing any issues.
  - A prompt in a language other than Thai or English is sent to the gateway, and the middleware does not attempt to translate it.

**4. Risk Register**

- **Risk:** The translation service is unavailable or returns incorrect translations.
  - **Detection:** Regularly test the translation service and monitor its response times. Implement retries with exponential backoff and circuit breakers to handle temporary unavailability.
- **Risk:** The language detection heuristic fails to accurately identify the language of the prompt.
  - **Detection:** Implement unit tests to validate the language detection heuristic. Monitor the logs for prompts that are not translated as expected and investigate these cases.
- **Risk:** The middleware causes a performance bottleneck due to the translation service's latency.
  - **Detection:** Monitor the gateway's performance and response times. Implement caching for frequently translated prompts to reduce the number of API calls to the translation service.
