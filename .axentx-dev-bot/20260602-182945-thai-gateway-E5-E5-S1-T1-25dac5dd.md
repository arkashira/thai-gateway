## Diagnosis  
The PRD requires Prometheus metrics for request count, latency, provider quota, and health status.  
The repository currently has no metric definitions, so the `/metrics` endpoint is empty and the unit tests that check for metric registration fail.  
We need to:

1. Define the four metrics (`total_requests`, `request_latency_seconds`, `provider_quota_remaining`, `health_status`).  
2. Register them with Prometheus’ default registry.  
3. (Optional but useful) expose a helper to wire the `/metrics` endpoint into the router.

## Proposed change  
Create a new file `metrics/metrics.go` that:

* Declares the metrics as exported variables.  
* Registers them in an `init()` function.  
* Provides a `RegisterMetrics` helper that attaches the Prometheus handler to a Gin router.

No other files need to be touched for the metrics to be available; the existing `/metrics` route (if present) will automatically use the registered metrics.

## Implementation  

```diff
--- /dev/null
+++ /opt/axentx/thai-gateway/src/metrics/metrics.go
+package metrics
+
+import (
+	"github.com/gin-gonic/gin"
+	"github.com/prometheus/client_golang/prometheus"
+	"github.com/prometheus/client_golang/prometheus/promhttp"
+)
+
+// TotalRequests counts every request received by the gateway.
+var TotalRequests = prometheus.NewCounter(prometheus.CounterOpts{
+	Name: "total_requests",
+	Help: "Total number of requests received by the gateway.",
+})
+
+// RequestLatency records the processing latency of each request.
+var RequestLatency = prometheus.NewHistogram(prometheus.HistogramOpts{
+	Name:    "request_latency_seconds",
+	Help:    "Histogram of request processing latency in seconds.",
+	Buckets: prometheus.DefBuckets,
+})
+
+// ProviderQuotaRemaining tracks the remaining quota for each AI provider.
+// The provider name is exposed as a label.
+var ProviderQuotaRemaining = prometheus.NewGaugeVec(
+	prometheus.GaugeOpts{
+		Name: "provider_quota_remaining",
+		Help: "Remaining quota for each provider.",
+	},
+	[]string{"provider"},
+)
+
+// HealthStatus indicates gateway health: 1 = healthy, 0 = unhealthy.
+var HealthStatus = prometheus.NewGauge(prometheus.GaugeOpts{
+	Name: "health_status",
+	Help: "Health status of the gateway: 1 for healthy, 0 for unhealthy.",
+})
+
+// init registers all metrics with the default Prometheus registry.
+func init() {
+	prometheus.MustRegister(
+		TotalRequests,
+		RequestLatency,
+		ProviderQuotaRemaining,
+		HealthStatus,
+	)
+}
+
+// RegisterMetrics attaches the Prometheus HTTP handler to the provided Gin router.
+// It is a convenience helper; the application can call this during startup.
+func RegisterMetrics(r *gin.Engine) {
+	r.GET("/metrics", gin.WrapH(promhttp.Handler()))
+}
```
