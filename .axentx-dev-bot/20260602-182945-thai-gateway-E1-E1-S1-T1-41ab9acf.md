---

## 2. Core Design Principles

| Principle | Why it matters | How we implement it |
|-----------|----------------|---------------------|
| **Externalised configuration** | Enables runtime changes without recompilation. | Load a JSON/YAML file at startup. |
| **Strong validation** | Prevent silent failures. | Validate every required field; abort startup on error. |
| **Explicit logging** | Operators know what’s running. | Log a concise summary after successful load. |
| **Extensibility** | Future providers or fields can be added with minimal code changes. | Use struct tags and a slice of `Provider` structs. |
| **Testability** | Confidence that the loader works. | Unit tests for `LoadProviderConfig`. |
| **Safety** | Avoid race conditions when the config is accessed concurrently. | Store the config in a thread‑safe global or inject via context. |

---
