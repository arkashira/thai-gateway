### 1. Acceptance Criteria

- `GET /health` returns HTTP 200 with JSON body `{"status":"ok","providers":[...]}` within 100 ms under nominal load
- Each provider entry in the response includes `name`, `healthy` (bool), and `last_checked` (RFC3339 timestamp)

- A provider failing two consecutive pings transitions to `healthy: false` and is excluded from routing decisions immediately

- `GET /health` requires **no authentication** (public liveness probe)
