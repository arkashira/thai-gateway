## Unified, Production‑Ready Rate‑Limiting Middleware for the Thai‑Gateway

Below is a single, self‑contained implementation that combines the most robust ideas from the two candidates:

* **Per‑provider quota stored in Redis with a TTL** – guarantees that each provider’s rate‑limit window is independent and automatically resets.
* **Dynamic provider selection** – the gateway automatically falls back to the next provider when the current one is exhausted.
* **Correct 429 handling** – when *all* providers are exhausted the gateway returns a `429 Too Many Requests` with a `Retry‑After` header that reflects the remaining time until the next quota window.
* **FastAPI‑friendly** – the middleware is written as an ASGI middleware so it can be dropped into any FastAPI app without changing the route handlers.

> **Why this version is better than the candidates**  
> * Candidate 1 used a decorator on the route, which makes the provider selection logic hard to reuse and test.  
> * Candidate 2 was written in JavaScript and omitted the TTL logic for the quota key.  
> * The unified version keeps the logic in a single place (middleware), uses a strongly‑typed Redis client, and exposes a clean API (`request.state.provider`) for downstream handlers.

### 1. Redis Client (`redis_client.py`)

# /opt/axentx/thai-gateway/utils/redis_client.py
import redis
from typing import Optional

class RedisClient:
    """
    Thin wrapper around redis.StrictRedis that knows how to store
    per‑provider rate‑limit counters with a TTL.
    """

    def __init__(self, host: str = "localhost", port: int = 6379, db: int = 0):
        self._redis = redis.StrictRedis(
            host=host, port=port, db=db, decode_responses=True
        )

    def _key(self, provider: str) -> str:
        return f"rate_limit:{provider}"

    def get_remaining(self, provider: str) -> int:
        """Return the remaining quota for *provider* (0 if key missing)."""
        value = self._redis.get(self._key(provider))
        return int(value) if value is not None else 0

    def decrement(self, provider: str, ttl: int) -> None:
        """
        Atomically decrement the counter and reset the TTL.
        The counter is never allowed to go below 0.
        """
        pipe = self._redis.pipeline()
        pipe.watch(self._key(provider))
        try:
            current = int(pipe.get(self._key(provider)) or 0)
            if current <= 0:
                pipe.unwatch()
                return
            pipe.multi()
            pipe.decr(self._key(provider))
            pipe.expire(self._key(provider), ttl)
            pipe.execute()
        except redis.WatchError:
            # Another client modified the key – retry is not required here
            pass

    def reset(self, provider: str, quota: int, ttl: int) -> None:
        """Set the counter to *quota* and set the TTL."""
        self._redis.setex(self._key(provider), ttl, quota)

    def ttl(self, provider: str) -> int:
        """Return the remaining TTL for *provider* in seconds."""
        return self._redis.ttl(self._key(provider))
```
