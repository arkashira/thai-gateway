2. **YAML config parsing** – When a `config.yaml` file is present, the gateway merges its values with env‑vars, giving precedence to env‑vars for overlapping keys (e.g., `translation_api_key`).
3. **Docker image builds** – `docker build -t thai-gateway .` completes in ≤ 2 min, produces an image that runs the gateway on port 8080 by default, and respects the same env‑var/YAML behavior inside the container.
4. **README onboarding** – The README contains a step‑by‑step guide (install Docker, build image, run container with example env‑vars and optional `-v ./config.yaml:/app/config.yaml` mount) and the guide is validated by a script that checks for required sections (Installation, Build, Run, Configuration).
5. **Graceful shutdown** – Sending SIGTERM to the container triggers a clean shutdown (no hanging processes, logs “Shutting down”).
6. **Rate‑limit enforcement** – With `THAI_GATEWAY_RATE_LIMIT=5` the gateway allows at most 5 requests per second per provider; excess requests receive HTTP 429.
7. **Port exposure** – The Dockerfile exposes port 8080 and the container can be started with `-p 8080:8080` and is reachable from the host.
