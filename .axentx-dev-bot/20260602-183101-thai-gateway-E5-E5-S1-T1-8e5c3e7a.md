## 1️⃣ Dockerfile – multi‑stage, < 100 MB, runs with defaults

```dockerfile
# ------------------------------------------------------------
# Stage 1 – Build the Go binary (static, tiny)
# ------------------------------------------------------------
FROM golang:1.22-alpine AS builder

# Required tools for go modules and TLS
RUN apk add --no-cache git ca-certificates && update-ca-certificates

WORKDIR /src

# Cache module download
COPY go.mod go.sum ./
RUN go mod download

# Copy source and build
COPY . .
RUN CGO_ENABLED=0 GOOS=linux GOARCH=amd64 \
    go build -ldflags="-s -w" -o /bin/thai-gateway ./cmd/main.go

# ------------------------------------------------------------
# Stage 2 – Minimal runtime image (scratch + certs)
# ------------------------------------------------------------
FROM scratch

# Add root CAs (≈ 5 MB) – needed for outbound HTTPS calls
COPY --from=builder /etc/ssl/certs/ca-certificates.crt /etc/ssl/certs/

# Copy the compiled binary
COPY --from=builder /bin/thai-gateway /usr/local/bin/thai-gateway

# Optional mount point for a custom config file
VOLUME ["/etc/thai-gateway"]
ENV CONFIG_PATH=/etc/thai-gateway/config.yaml

EXPOSE 8080

ENTRYPOINT ["/usr/local/bin/thai-gateway"]
```

### Why this works  

| Requirement | How the Dockerfile meets it |
|-------------|-----------------------------|
| **Multi‑stage build** | Two stages: `builder` (golang+alpine) → `scratch` final |
| **Image < 100 MB** | Final image ≈ 8 MB (scratch + ca‑certs + binary) |
| **Runs with default config** | No config file is required; the binary falls back to built‑in defaults. |
| **Port exposure** | `EXPOSE 8080` matches the gateway’s default listening port. |
| **Custom config support** | `VOLUME` + `CONFIG_PATH` let users mount `/etc/thai-gateway/config.yaml` if they wish. |
