# Task: Set up Winston logging for requests

## Diagnosis
The current implementation does not have any logging mechanism for requests. To address this, we need to integrate Winston, a popular logging library in Node.js, to log all requests with timestamps and provider routing decisions.

## Proposed change
We will create a new file `/opt/axentx/thai-gateway/src/logger.js` to set up Winston logging. Then, we will integrate this logger in the main gateway file `/opt/axentx/thai-gateway/src/index.js` to log all incoming requests.

## Implementation

1. First, install Winston using npm:
```bash
npm install winston
```

2. Create a new file `/opt/axentx/thai-gateway/src/logger.js` and add the following code:

```javascript
// /opt/axentx/thai-gateway/src/logger.js
const winston = require('winston');

const logger = winston.createLogger({
  level: 'info',
  format: winston.format.combine(
    winston.format.timestamp(),
    winston.format.json()
  ),
  transports: [
    new winston.transports.Console(),
    new winston.transports.File({ filename: 'error.log', level: 'error' }),
    new winston.transports.File({ filename: 'combined.log' })
  ]
});

module.exports = logger;
```

3. In `/opt/axentx/thai-gateway/src/index.js`, import the logger and log all incoming requests:

```javascript
// /opt/axentx/thai-gateway/src/index.js
const express = require('express');
const logger = require('./logger');

const app = express();

app.use((req, res, next) => {
  logger.info({
    method: req.method,
    url: req.url,
    timestamp: new Date(),
    body: req.body
  });
  next();
});

// Rest of your application...
```

## Verification
To verify the implementation, send a request to your gateway and check the logs in the `combined.log` file. You should see the logged request with timestamps and the request body.
