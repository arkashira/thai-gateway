## TL;DR  
Add a **single, well‑tested configuration package** that  

1. **Parses** a `providers.yaml` file into a typed Go struct (providers + translation settings).  
2. **Validates** the content and returns **clear, user‑friendly errors** when the file is missing, malformed, or fails structural checks.  
3. **Watches** the file with `fsnotify` and **reloads it atomically** (no restart required).  
4. Exposes the current configuration through a **thread‑safe getter** that the rest of the gateway can use.  

All changes are limited to three new/modified files:

| File | Purpose |
|------|---------|
| `config/config.go` | Full implementation (structs, loader, validator, watcher, atomic accessor). |
| `config/providers.yaml` | Example configuration that matches the struct. |
| `cmd/main.go` (or wherever the server is started) | Initialise the config package and start the watcher. |

The solution merges the strongest points of both proposals: the **concise struct design** from Candidate 2, the **dynamic‑reload loop** from Candidate 1, and adds **mutex‑protected access** plus **structured validation** for robustness.

---

## 1. Design Overview  

| Concern | Decision |
|---------|----------|
| **File format** | YAML (human‑readable, already used in the PRD). |
| **Parsing library** | `gopkg.in/yaml.v3` – the de‑facto standard, supports strict unmarshalling. |
| **Validation** | Simple hand‑rolled checks (required fields, URL format) – no extra dependency, but easy to replace with a validator library later. |
| **Reload trigger** | `fsnotify` watches the file for `Write`/`Create`/`Rename` events. |
| **Thread safety** | A `sync.RWMutex` protects a package‑level `*Config` pointer; callers read via `config.Current()`. |
| **Error reporting** | All errors are wrapped with context (`fmt.Errorf("…: %w", err)`) and logged with `log.Printf`. The loader returns the error to the caller so the process can abort on start‑up failures. |
| **Zero‑downtime** | The watcher replaces the internal pointer only after a successful load, so the rest of the code always sees a *valid* configuration. |
