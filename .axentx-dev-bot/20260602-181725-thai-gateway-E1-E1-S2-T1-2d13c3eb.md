**Feature Implementation: Safety Compliance Guardian (SCG)**  

**1. Overview**  
SCG is a lightweight, pluggable agent that scans source code during CI/CD runs, detects regulatory or safety violations, and auto‑generates remediation suggestions. It uses a vectorized knowledge base of legal and industry safety requirements (e.g., GDPR, ISO 26262, HIPAA) stored in a pgvector index. The agent is exposed as a GitHub Action and a CLI tool, enabling integration into existing pipelines.

**2. Architecture**  

```
┌───────────────────────┐
│  CI/CD Pipeline (GitHub Actions) │
└───────────────┬───────┘
                │
                ▼
┌───────────────────────┐
│  SCG Runner (Python)  │
│  ├─ Code Scanner      │
│  ├─ Vector Search     │
│  ├─ Violation Matcher │
│  ├─ Remediation Engine│
│  └─ Reporter          │
└───────────────┬───────┘

```

**3. Key Components**

| Component | Responsibility | Implementation Highlights |
|-----------|----------------|---------------------------|
| **Code Scanner** | Parses Python, JavaScript, and C/C++ files; extracts identifiers, comments, and API calls. | Uses `ast` for Python, `esprima` for JS, `clang.cindex` for C/C++. |
| **Vector Search** | Embeds extracted code snippets and queries the pgvector index. | Uses `sentence-transformers` model `all-MiniLM-L6-v2`; similarity threshold 0.75. |
| **Violation Matcher** | Maps similarity hits to specific safety rules (e.g., “No hard‑coded passwords”). | Rule metadata stored in `rules.json`. |
| **Remediation Engine** | Generates context‑aware code patches or documentation links. | Leverages OpenAI Codex API (or local LLM) with prompt templates. |
| **Reporter** | Emits SARIF format, GitHub check run, and optional Slack notification. | Uses `sarif-om` library. |
