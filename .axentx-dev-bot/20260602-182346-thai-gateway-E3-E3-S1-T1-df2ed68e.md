## 1. Acceptance Criteria
1. **English detection** – The gateway correctly identifies an English‑only prompt with ≥ 90 % precision and ≥ 95 % recall on a mixed‑language test set.
2. **Thai translation** – Detected English prompts are translated to Thai using the `ThaiTranslationModel` and the returned text is valid UTF‑8 Thai script.
3. **Latency** – End‑to‑end translation (detection → model inference → response) completes in **≤ 200 ms** for 95 % of requests on a typical production‑sized payload (≤ 256 bytes).
4. **Audit logging** – The original English prompt is logged at `INFO` level with a unique request ID and timestamp; the log entry must be searchable and contain no truncation.
5. **Fallback behavior** – If detection fails or the model returns an error, the original English prompt is forwarded unchanged and a warning is logged.
6. **Concurrency safety** – The translator service handles at least 100 concurrent requests without race conditions or deadlocks.
7. **Observability** – Metrics `translation_latency_ms` and `translation_errors_total` are emitted to Prometheus (or equivalent) with correct labels.
