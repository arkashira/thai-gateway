### 1. Acceptance Criteria
1. **YAML schema compliance** – `gateway.yaml` must contain a top‑level `providers` array; each element must have **exactly** the fields `id` (string), `url` (valid URL), `api_key` (non‑empty string), and `quota` (positive integer).  
2. **Successful load** – On application start, the loader reads `gateway.yaml` and returns a `Config` object containing all providers without panicking.  
3. **Missing‑field logging** – If any provider entry lacks a required field, the loader logs a warning/error message that includes the provider index and the missing field name.  
4. **Invalid‑value handling** – Invalid values (e.g., malformed URL, non‑integer quota, non‑positive quota) cause the loader to reject the config and exit with a non‑zero status, logging a clear error.  
5. **Sample config presence** – The repository contains `gateway.yaml.example` (or similar) that validates against the same schema.  
6. **Graceful fallback** – When the config file is absent, the loader returns a default empty provider list and logs a warning.

mod tests {
    use super::*;
    use std::fs;
    use tempfile::NamedTempFile;

    // Helper to write temporary yaml content
    fn write_yaml(content: &str) -> NamedTempFile {
        let mut file = NamedTempFile::new().unwrap();
        fs::write(file.path(), content).unwrap();
        file
    }

    #[test]
    fn loads_valid_config() {
        let yaml = r#"
providers:
  - id: "provider1"
    url: "https://api.example.com"
    api_key: "secret123"
    quota: 1000
  - id: "provider2"
    url: "http://api2.example.org"
    api_key: "key456"
    quota: 500
"#;
        let file = write_yaml(yaml);
        let cfg = Config::load_from_path(file.path()).expect("should load");
        assert_eq!(cfg.providers.len(), 2);
        assert_eq!(cfg.providers[0].id, "provider1");
        assert_eq!(cfg.providers[1].quota, 500);
    }

    #[test]
    fn rejects_missing_field() {
        let yaml = r#"
providers:
  - id: "p1"
    url: "https://api.example.com"

        let err = Config::load_from_path(file.path()).unwrap_err();
