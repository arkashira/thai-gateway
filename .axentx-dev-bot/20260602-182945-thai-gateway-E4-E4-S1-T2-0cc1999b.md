```diff
+// Provider is the abstraction for any downstream AI service.
+type Provider interface {
+	// Call forwards the request payload to the provider and returns the raw response body.
+	Call(ctx context.Context, payload []byte) (*http.Response, error)
+}
+
+// MockProvider is a very small stub that echoes the incoming payload.
+// It satisfies the Provider interface and is sufficient for the current
+// integration test / manual verification.
+type MockProvider struct{}
+
+func (m *MockProvider) Call(ctx context.Context, payload []byte) (*http.Response, error) {
+	// Echo the payload back as JSON with a 200 status.
+	// In a real implementation this would perform an HTTP request to the
+	// selected AI vendor.
+	rec := &http.Response{
+		StatusCode: http.StatusOK,
+		Body:       nopCloser{Reader: json.RawMessage(payload)},
+		Header:     make(http.Header),
+	}
+	rec.Header.Set("Content-Type", "application/json")
+	return rec, nil
+}
+
+// nopCloser implements io.ReadCloser for a static byte slice.
+type nopCloser struct {
+	Reader json.RawMessage
+}
+
+func (nc nopCloser) Read(p []byte) (int, error) { return copy(p, nc.Reader), nil }
+func (nc nopCloser) Close() error               { return nil }
```

### 1. Acceptance Criteria
1. **Endpoint registration** – `POST /chat` returns **200** for a well‑formed request and **405** for non‑POST methods.  
2. **Payload validation** – Request body must be valid JSON containing at least `prompt` (string) and `model` (string); missing or malformed fields yield **400** with an error message.  
3. **Middleware execution order** – `translateMiddleware` → `routingMiddleware` → `forwardMiddleware` are each invoked exactly once per request.  
4. **Translation middleware** – Adds a `translatedPrompt` field to the request context; the value must be the original `prompt` upper‑cased (simple deterministic rule for test).  
5. **Routing middleware** – Selects a provider based on `model`; for tests the `MockProvider` is always chosen and its name is stored in the context.  
6. **Forward middleware** – Calls the selected provider’s `Handle(ctx, req)` method and writes the provider’s raw HTTP response body back to the client unchanged.  
7. **Error handling** – Any error in a middleware returns a JSON error response with appropriate HTTP status (e.g., 400 for validation, 500 for internal failures) and aborts further middleware execution.
