## Unified, Production‑Ready Translation Gateway

Below is a single, self‑contained implementation that pulls together the strongest ideas from the three proposals:

| Requirement | What we do | Why it works |
|-------------|------------|--------------|
| **Configurable translation service** | Load a JSON config (`config.json`) that specifies the service name, endpoint, and any auth needed. | Keeps the gateway agnostic to the provider and allows runtime changes without code changes. |
| **Logging** | Use the standard `log` package (or a structured logger like `zap`) to record the original prompt, translated prompt, and any errors. | Provides traceability for debugging and compliance. |
| **Sync & async workflows** | Expose two HTTP handlers: `/translate` (sync) and `/translate/async` (async). The async version returns immediately with a job ID and stores the result in a thread‑safe map. | Gives callers the flexibility to choose latency vs. throughput. |
| **Fallback to original prompt** | If the translation call fails, the gateway logs the error and returns the original prompt unchanged. | Guarantees that the downstream service always receives a prompt, even if translation is unavailable. |
| **Retry & circuit‑breaker** | Wrap the HTTP client with a retry policy (up to 3 attempts) and a simple circuit‑breaker that opens after 5 consecutive failures. | Increases resilience against transient network or provider issues. |
| **Extensibility** | The `Translator` interface allows adding new providers (e.g., Azure, DeepL) without touching the middleware. | Future‑proofs the codebase. |

	}

}

// translator/translator.go
package translator

import (
	"bytes"
	"context"
	"encoding/json"
	"fmt"
	"io"
	"net/http"
	"sync"
	"time"
)

type Translator interface {
	Translate(ctx context.Context, prompt string) (string, error)
}

// ---------- Google Translator ----------
type googleTranslator struct {
	endpoint string
	apiKey   string
	client   *http.Client
	mu       sync.Mutex // protects circuit breaker state
	// simple circuit breaker
	consecutiveFails int
	circuitOpen      bool
}

func NewGoogleTranslator(endpoint, apiKey string) *googleTranslator {
	return &googleTranslator{
		endpoint: endpoint,
		apiKey:   apiKey,
		client: &http.Client{
			Timeout: 10 * time.Second,
		},
	}
}

func (g *googleTranslator) Translate(ctx context.Context, prompt string) (string, error) {
	g.mu.Lock()
	if g.circuitOpen {
		g.mu.Unlock()
		return "", fmt.Errorf("circuit open")
	}
	g.mu.Unlock()

	// Build request
	body := map[string]interface{}{
		"q":      prompt,
		"target": "th",
	}
	if g.apiKey != "" {
		body["key"] = g.apiKey
	}
	b, _ := json.Marshal(body)

	req, err := http.NewRequestWithContext(ctx, http.MethodPost, g.endpoint, bytes.NewReader(b))
	if err != nil {
		return "", err
	}
	req.Header.Set("Content-Type", "application/json")

	// Retry loop
	var resp *http.Response
	for i := 0; i < 3; i++ {
		resp, err = g.client.Do(req)
		if err == nil && resp.StatusCode == http.StatusOK {
			break
		}
		if resp != nil {
			resp.Body.Close()
		}
		time.Sleep(200 * time.Millisecond)
	}
	if err != nil {
		g.recordFailure()
		return "", err
	}
